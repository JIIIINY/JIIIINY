# 배경 색상에 따라 바뀌는 텍스트 — 분석 & 재현

## 원본 효과 분석

영상에서 "A new dawn" 텍스트는 파랑·골드·오렌지·레드가 흐르는 그라데이션 위에 놓여 있고,
**텍스트 색이 바로 뒤 배경에 따라 실시간으로 변합니다.**

핵심 메커니즘 (프레임 추출 + 픽셀 측정으로 확인):

1. **색조(Hue)는 배경을 따라간다** — 텍스트는 항상 그 자리 배경의 색조와 비슷한 계열.
   파랑 위에선 청록빛, 노랑 위에선 황갈빛으로 물듭니다.
2. **밝기(Lightness)는 가독 방향으로 밀어낸다** —
   어두운 배경 → 흰색 쪽으로 밝게, 밝은 배경 → 같은 색조의 어두운 톤(tone-on-tone)으로.
   덕분에 어디서든 읽히면서도 배경과 어우러지는 "은은하게 빛나는" 느낌.
3. **픽셀 단위 적응** — 한 단어 안에서도 글자마다 뒤 배경이 다르므로 색이 갈립니다.
   (예: 색 경계에 걸친 단어는 앞 글자는 밝게, 뒷 글자는 어둡게 분리)
4. **채도 소폭 상승** — 측정상 텍스트가 배경보다 채도가 조금 높음 → 살짝 또렷.
5. **보조 아이콘(작은 링)** 은 더 극적인 **보색(Complement) 반전**: 노랑 위 → 파랑, 파랑 위 → 황색.

### 알고리즘 (의사코드)
```
배경픽셀 (r,g,b)에 대해:
  Lb = luminance(r,g,b)
  (h,s,l) = rgb→hsl(r,g,b)
  target   = (Lb < 0.5) ? 1.0(흰색) : 0.18(어두운 톤)
  strength = 0.55 * (0.7 + 0.6*|Lb-0.5|*2)      # 극단일수록 강하게
  L' = lerp(l, target, strength)                 # 밝기만 가독 방향으로
  S' = min(s*1.25 + 0.12, 1)                      # 채도 소폭 부스트
  텍스트색 = hsl→rgb(h, S', L')                   # 색조 h는 유지
```

## 재현 방식
- 배경: 6색 팔레트(파랑/네이비/골드/오렌지/레드/밝은파랑)를 천천히 순환하는 4개의 이동 색 필드.
  날카로운 falloff로 각 구역의 색을 순수하게 유지하고 경계에서만 부드럽게 섞어 원본의 흐르는 룩 재현.
- 텍스트: 2배 슈퍼샘플링(720×1600)으로 글자 마스크를 만들고, 위 알고리즘으로 **픽셀별 적응 색** 합성 후 360×800으로 다운스케일.
- 키오스크 크롬: 상태바(시간/배터리/신호), "Tap to begin", 홈 인디케이터, 디바이스 베젤.

## 인터랙티브 데모
`adaptive-text-kiosk.html` 을 브라우저에서 열면 실시간으로 조절 가능:
- **문구** 변경 · **대비 강도** · **채도 부스트** · **배경 속도**
- **적응 방식**: Luminance(원본 방식) / Complement(보색) / Static White(비교용)
